## 🤝🏼 Collaboration

### Branch Convention

Postfix feature branch with issue #.

### Branch

> `main` : 최종 버전  
> `release` : 배포   
> `dev` : Backend 개발용   
> `feature` : 기능 개발용   
ex. `feature/#26-login` : commit title
`fix` : 버그 수정용   
ex. `fix/#30-login` : commit title
`refactor` : 리팩토링용   
ex. `refactor/#32-login` : commit title

#### Merge
> `→ dev` : Squash Merge   
> `나머지` : Default Merge   

### Commit Convention

Prefix with Issue #    
Ex: {커밋 타입} #{이슈 번호}: {커밋 제목}    

- {커밋 본문} (선택 사항)      

> `feat` : 새로운 기능 추가   
> `fix` : 버그 수정   
> `refactor` : 코드 리팩토링   
> `test` : 테스트 코드   
> `docs` : 문서 수정   
> `build` : 빌드 관련 파일     
> `chore` : 그 외 자잘한 수정
